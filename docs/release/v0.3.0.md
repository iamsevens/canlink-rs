# v0.3.0 Release Notes

发布日期：2026-03-25

## 亮点
- 新增周期消息发送与 ISO-TP 传输能力（对应 004 规范）
- CLI 新增 periodic 与 isotp 相关指令
- 示例与文档更新，补齐 API 与用户指南

## 新增
- 周期发送（periodic）
- `PeriodicMessage` / `PeriodicScheduler` / `PeriodicStats`
- 1ms–10000ms 间隔，支持动态调整数据与间隔
- 支持至少 32 并发周期消息
- 出错时跳过本次、不中断后续周期
- ISO-TP（isotp）
- `IsoTpChannel`、`IsoTpConfig`、`IsoTpFrame`、`IsoTpError`
- 自动分段与重组、流控处理、序列号校验
- 可配置超时、FC(Wait)、支持 `abort()`
- 支持进度回调 `IsoTpCallback`
- 高级能力
- `AddressingMode`（Normal/Extended/Mixed）
- `FrameSize`（Auto/Classic8/Fd64）
- `StMin`（0–127ms / 100–900μs）、`FlowStatus`
- 可配置 padding 与最大缓冲区

## CLI 扩展
- `canlink send --periodic <ms> --count <n>`
- `canlink isotp send`
- `canlink isotp receive`
- `canlink isotp exchange`

## 示例
- `periodic_send.rs`
- `isotp_transfer.rs`

## 变更
- Feature flags 调整：`periodic` / `isotp` 依赖 `async-tokio`
- `full` feature 包含 `periodic` / `isotp`

## 性能与验收
- 周期发送抖动 < 5ms（10ms 间隔）
- ISO-TP 吞吐优化，STmin 可调
- SC-001~SC-005 均满足（SC-005：线覆盖 89.37%，区域覆盖 90.51%）

## 技术细节
- 新模块：`canlink_hal::periodic`、`canlink_hal::isotp`
- 新增测试：ISO-TP 37 个、周期 24 个
- 无新增外部依赖
