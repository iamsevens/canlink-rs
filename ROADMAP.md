## 🎯 核心理念

**多后端支持策略**: 将不同的 CAN 工具作为独立的硬件后端实现，通过统一的抽象层接口进行访问。

```
应用层
   ↓
CAN 硬件抽象层 (002)
   ↓
┌─────────────┬─────────────┬─────────────┬─────────────┬──────┐
│ LibTSCAN    │ TSMaster    │ Vehicle Spy │ CANoe       │ Mock │
│ (P1)        │ (P2)        │ (P3)        │ (P3)        │ (P1) │
└─────────────┴─────────────┴─────────────┴─────────────┴──────┘
```

**版本计划**: v0.2.0
**状态**: 未来计划

**设计决策** (来自 research.md):

**用例**:

- 事件驱动架构
- 与异步框架集成
- 非阻塞 I/O

**依赖**:
- 同步 API 稳定 (v0.1.0)
