# Contributing

Thank you for your interest in contributing.

## Basics

- Use UTF-8 for all files.
- Run `cargo fmt` and `scripts\check.bat` (Windows) or `./scripts/check.sh` (Unix).
- Add tests where applicable and ensure existing tests pass.
- Specs and plans live under `specs/` (do not put spec/plan/task under `docs/`).

## Reporting Issues

- Provide minimal repro steps, OS, Rust version, and hardware/runtime details.
- If the issue depends on LibTSCAN, include your DLL/LIB version and device model.
